# Home Media Server

Домашний медиасервер на базе Torrent + DLNA + SMB.

Скрипт устанавливает на Ubuntu/Debian:

| Сервис | Назначение |
|---|---|
| Transmission | Торрент-клиент с веб-интерфейсом и watch-папкой |
| MiniDLNA (ReadyMedia) | Раздача видео, музыки и фотографий по DLNA |
| Samba | Сетевые папки SMB для Windows, macOS и Linux |

## Установка одной командой

```bash
sudo bash -c 'set -Eeuo pipefail; apt-get update; apt-get install -y curl ca-certificates; tmp=$(mktemp); trap "rm -f $tmp" EXIT; curl -fsSL https://raw.githubusercontent.com/Tuma58/home-media-server/main/install.sh -o "$tmp"; bash "$tmp"'
```

Команда устанавливает `curl`, скачивает актуальный скрипт во временный файл, запускает его и удаляет временный файл. Сам скрипт сохраняет команду управления `/usr/local/sbin/home-media-server`.

При первой установке создаются каталоги, пользователи, конфиги и правила UFW (только если UFW уже активен). Если пароли не переданы через окружение, скрипт генерирует случайные пароли и один раз показывает их в итоговой сводке.

Поддерживаются Ubuntu 20.04+ и Debian 11+.

## Повторный запуск и меню управления

```bash
sudo home-media-server
```

Повторный запуск не переустанавливает пакеты и не перезаписывает конфиги. Он открывает меню:

1. SMB: общие папки — просмотр, добавление и удаление управляемых шар, проверка `smb.conf`.
2. SMB: пользователи — просмотр, добавление, смена пароля, включение, отключение и удаление SMB-учётной записи.
3. DLNA — статус, запуск, остановка, перезапуск, перестроение индекса, медиакаталоги и имя сервера.
4. Статус сервисов.

Удаление шары или DLNA-каталога из меню удаляет только запись из конфигурации. Файлы и Linux-пользователи не удаляются.

Основная шара `Media` разрешена группе `@media-share`, поэтому добавленные через меню SMB-пользователи сразу получают к ней доступ. Для дополнительной шары можно указать конкретных пользователей или оставить поле пустым, чтобы использовать ту же общую группу.

Дополнительные команды:

```bash
sudo home-media-server --menu
sudo home-media-server --status
home-media-server --help
```

## Настройка первой установки

Параметры можно передать через переменные окружения:

```bash
sudo MEDIA_ROOT=/data/media \
  TRANSMISSION_USER=admin \
  TRANSMISSION_PASS='сложный-пароль-1' \
  SAMBA_USER=media \
  SAMBA_PASS='сложный-пароль-2' \
  bash install.sh
```

| Переменная | По умолчанию | Описание |
|---|---|---|
| `MEDIA_ROOT` | `/srv/media` | Корень медиакаталогов |
| `MEDIA_GROUP` | `media-share` | Общая Linux-группа для записи |
| `TORRENT_USER` | `debian-transmission` | Системный пользователь Transmission |
| `TRANSMISSION_PORT` | `9091` | Порт веб-интерфейса |
| `TRANSMISSION_USER` | `admin` | Логин Transmission |
| `TRANSMISSION_PASS` | случайный | Пароль Transmission |
| `DLNA_NAME` | `HomeMedia` | Имя DLNA-сервера |
| `DLNA_PORT` | `8200` | HTTP-порт MiniDLNA |
| `SAMBA_USER` | `media` | Первый SMB-пользователь |
| `SAMBA_PASS` | случайный | Пароль первого SMB-пользователя |
| `SAMBA_SHARE_NAME` | `Media` | Имя первой SMB-шары |

Переменные первой установки не применяются заново при обычном повторном запуске. Дальнейшие изменения выполняются через меню или непосредственно в конфигурации сервисов.

## Каталоги

```text
/srv/media/
├── downloads/   # завершённые загрузки
├── incomplete/  # незавершённые загрузки
├── watch/       # .torrent-файлы для автодобавления
├── video/       # видео DLNA
├── music/       # музыка DLNA
└── photo/       # фотографии DLNA
```

Каталоги получают общую группу `media-share`, setgid-бит и права групповой записи. Это позволяет Transmission и авторизованным SMB-пользователям работать с одними файлами без небезопасного режима `0777`.

## Где хранятся настройки

- `/etc/transmission-daemon/settings.json` — Transmission;
- `/etc/minidlna.conf` — MiniDLNA;
- `/etc/samba/smb.conf` — основной конфиг Samba;
- `/etc/samba/smb.conf.d/torrent-dlna-shares.conf` — шары, созданные этим скриптом;
- `/etc/torrent-dlna/installed` — маркер завершённой установки.

Перед первым изменением существующего конфига создаётся резервная копия с суффиксом `.torrent-dlna.bak`. Повторные запуски её не перезаписывают.

Если найдена только часть сервисов, скрипт не считает это готовой установкой и просит подтверждение перед настройкой существующих конфигов. Для заранее одобренного неинтерактивного запуска можно передать `ALLOW_RECONFIGURE=1`.

## Доступ после установки

```text
Transmission: http://IP-СЕРВЕРА:9091
Windows SMB:  \\IP-СЕРВЕРА\Media
macOS SMB:    smb://IP-СЕРВЕРА/Media
```

DLNA-сервер появляется в совместимых телевизорах и плеерах под именем `HomeMedia` (или значением `DLNA_NAME`).

## Важные замечания по безопасности

- Не публикуйте порты Transmission, Samba и DLNA напрямую в интернет.
- Используйте случайные уникальные пароли; пароли по умолчанию `admin/admin` и `media/media` больше не задаются.
- Меню проверяет имена, пути и конфиг Samba через `testparm` до применения; при ошибке изменение откатывается.
- Для SMB используется минимум протокол SMB2 из системного конфига Samba. При миграции со старой версии существующий основной `smb.conf` сохраняется, поэтому проверьте его параметры отдельно через пункт меню.
